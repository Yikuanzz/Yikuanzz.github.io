异步 IO

在实际执行 IO 操作的时候，我们的程序会进入阻塞去等待 IO 操作的完成。这样会带来的后果是，要花费大量的时间来等待，浪费了 CPU 的执行效率。

所以，我们会使用多线程或者多进程来并发执行代码，就是说我们每个用户分配一个线程，让 IO 操作在线程中执行。

但是，因为线程的切换也是有不小的开销，所以一旦线程数量过多，那么 CPU 的时间就会花费在线程的切换上了，这样程序的性能就会下降。

于是，另一种方法是异步 IO，当我们需要执行一个很耗时的 IO 操作时，并不等待 IO 结果，而是在 IO 执行完后再去处理。

协程

协程（Coroutine）是一种并发模型，它允许程序在单线程内实现多个独立的执行线程，这些线程可以非阻塞地挂起和恢复，以实现协作式多任务处理。

协程与线程之间的区别：

  • 1、资源管理：线程是操作系统的资源，而协程不调用操作系统，所以它也被称为用户态线程。
  • 2、调度方式：线程是抢占式的，由操作系统进行调度；而协程是非抢占式，需要用户释放使用权来切换到其他协程。
  • 3、并发和并行：线程是真正意义上的并行执行，而协程主要是在等待 IO 的过程中，重复使用线程。
  • 4、内存空间：线程共享同一块内存，而协程没有独立的内存空间。
  • 5、状态保留：协程能保留上一次调用时的状态。
  • 6、开销：线程的创建和销毁都是系统级的，协程的开销相对要小。

传统的生产者与消费者模型，就是一个线程写消息，另一个线程读消息，我们看看协程的实现方式：

def consumer():
    r = ''
    while True:
        n = yield r
        if not n:
            return
        print('[消费者] 正在消费 %s...' % n)
        r = '消费完成'


def produce(c):
    c.send(None)
    n = 0
    while n < 5:
        n = n + 1
        print('[生产者] 生产中 %s...' % n)
        r = c.send(n)       # 调用消费者开始生产
        print('[生产者] 消费者返回值: %s' % r)
    c.close()


if __name__ == '__main__':
    c = consumer()
    produce(c)

asyncio

asyncio 是用来编写并发代码的库，它提供了一组高层级的 API 用于 ：

  • 1、并发地运行 Python 协程。
  • 2、控制子进程执行网络 IO。
  • 3、通过队列实现分布式任务。
import asyncio

async def main():
    print('Hello ...')
    await asyncio.sleep(1)
    print('... World!')

asyncio.run(main())

我们再看一个例子，异步函数会被一个单独的事件循环使用，当读取到 await 的语句时，会通知事件循环先执行其他任务，在 await 语句中的内容执行完毕后再调用回该函数。

import asyncio
import time

async def count():
    print("1")
    await asyncio.sleep(1)
    print("2")

async def do():
    await asyncio.gather(count(), count(), count())

if __name__ == '__main__':
    s = time.perf_counter()
    asyncio.run(do())
    e = time.perf_counter() - s
    print(f"{__file__} 执行了 {e:0.2f} 秒")

通过上面的例子，我们知道 async def 会产生一个 coroutine 对象，并且 coroutine 函数中可以包含 await 关键字，当执行到它的时候，coroutine 会暂停执行而去执行其他任务，直到 await 任务执行完后再继续执行该 coroutine。

串联 Coroutines

我们知道 coroutine 对象能够被另一个 coroutine 通过 await 调用。

通过这个特性，我们可以将程序变成串联的调用链。

import asyncio
import random
import time


async def part1(n):
    i = random.randint(0, 10)
    print(f"({n}) 开始做 {i} 个馒头。")
    await asyncio.sleep(i)
    print(f"({n}) 做好了 {i} 个馒头。")
    return i


async def part2(n, arg):
    i = random.randint(0, 10)
    print(f"({n}) 开始吃馒头。")
    await asyncio.sleep(i)
    print(f"({n}) 花了 {i} 分钟吃了 {arg} 个馒头.")
    return i


async def chain(n):
    s = time.perf_counter()
    p1 = await part1(n)
    p2 = await part2(n, p1)
    e = time.perf_counter() - s
    print(f"-->{n}的工作 => 做面包用了{p1}分钟, 吃面包用了{p2}分钟.")


async def do(names):
    task = (chain(i) for i in names)
    await asyncio.gather(*task)


if __name__ == '__main__':
    p = ["张三", "李四", "王五"]
    s = time.perf_counter()
    asyncio.run(do(p))
    e = time.perf_counter() - s
    print(f"程序执行用时 {e:0.2f} 秒。")

队列 Coroutines

队列模式中，一些互相独立的生产者向队列中放入事务。每个生产者都可能在随机的时间向队列中放入任意多个事项，而一旦事项被放入队列，就会有一组消费者马上从队列中读取尽量多的事项来进行处理，无需等待任何信号来触发这一动作。

asyncio.queue 中定义了三种 coroutine：

  • await queue.put() 向队列放入事务，如果队列满了，就返回并等待到有新空间时再调用。
  • await queue.get() 从队列中取出事务，如果队列为空，则返回并等待到队列中有新的事务时再调用。
  • await queue.join() 判断队列中的所有事务是否被全部取出并处理，如果是的话就返回。
import asyncio
import itertools as it
import os
import time
import random


async def produce(name, q):
    n = random.randint(0, 10)
    for _ in it.repeat(None, n):
        await asyncio.sleep(1)
        i = os.urandom(5).hex()
        t = time.perf_counter()
        await q.put((i, t))
        print(f"[生产者] {name} 生产了 <{i}> 到队列中。")


async def consume(name, q):
    while True:
        await asyncio.sleep(1)
        i, t = await q.get()
        now = time.perf_counter()
        print(f"[消费者] {name} 消费了 <{i}> ，花费了 {now-t:0.5f} 秒。")
        q.task_done()


async def do(nProducer, nConsumer):
    q = asyncio.Queue()
    producers = [asyncio.create_task(produce(n, q)) for n in range(nProducer)]
    consumer = [asyncio.create_task(consume(n, q)) for n in range(nConsumer)]
    await asyncio.gather(*producers)
    await q.join()
    for c in consumer:
        c.cancel()


if __name__ == '__main__':
    s = time.perf_counter()
    asyncio.run(do(5, 5))
    e = time.perf_counter() - s
    print(f"程序执行用时 {e:0.5f} 秒。")

异步 HTTP 请求

这个程序的内容是这样的：

  1. 从一个本地文件 urls.txt 里读取一系列的 URL 地址
  2. 向这些 URL 发送 GET 请求然后解析返回值。如果失败，就不再继续处理这个 URL 了。
  3. 搜索返回的 HTTP 内容里包含的 href 里的 URL
  4. 将这些 URL 写入本地文件 foundurls.txt
  5. 尽量用异步和并发来实现上述逻辑。（使用 aiohttp 来发送 HTTP 请求，使用 aiofiles 来写入文件。这两个库是都是异步IO模式的很好的例子）。
$ cat urls.txt
https://regex101.com/
https://docs.python.org/3/this-url-will-404.html
https://www.nytimes.com/guides/
https://www.mediamatters.org/
https://1.1.1.1/
https://www.politico.com/tipsheets/morning-money
https://www.bloomberg.com/markets/economics
https://www.ietf.org/rfc/rfc2616.txt


""" 异步的获取嵌入在 HTML 内的 HTTP 链接 """

import asyncio
import logging
import re
import sys
from typing import IO
import urllib.error
import urllib.parse

import aiofiles
import aiohttp
from aiohttp import ClientSession

logging.basicConfig(
    format="%(asctime)s %(levelname)s:%(name)s: %(message)s",
    level=logging.DEBUG,
    datefmt="%H:%M:%S",
    stream=sys.stderr,
)
logger = logging.getLogger("areq")
logging.getLogger("chardet.charsetprober").disabled = True

HREF_RE = re.compile(r'href="(.*?)"')

async def fetch_html(url: str, session: ClientSession, **kwargs) -> str:
    """ 利用 HTTP GET 请求获取 HTML 页面。

    kwargs 作为参数传递给 `session.request()`.
    """

    resp = await session.request(method="GET", url=url, **kwargs)
    resp.raise_for_status()
    logger.info("Got response [%s] for URL: %s", resp.status, url)
    html = await resp.text()
    return html

async def parse(url: str, session: ClientSession, **kwargs) -> set:
    """ 获取 url 链接返回的 html 中内嵌的 href """
    found = set()
    try:
	html = await fetch_html(url=url, session=session, **kwargs)
    except (
	aiohttp.ClientError,
	aiohttp.http_exceptions.HttpProcessingError,
    ) as e:
	logger.error(
	    "aiohttp exception for %s [%s]: %s",
	    url,
	    getattr(e, "status", None),
	    getattr(e, "message", None),
	)
	return found
    except Exception as e:
	logger.exception(
	    "Non-aiohttp exception occured:  %s", getattr(e, "__dict__", {})
	)
	return found
    else:
	for link in HREF_RE.findall(html):
	    try:
		abslink = urllib.parse.urljoin(url, link)
	    except (urllib.error.URLError, ValueError):
		logger.exception("Error parsing URL: %s", link)
		pass
	    else:
		found.add(abslink)
	logger.info("Found %d links for %s", len(found), url)
	return found

async def write_one(file: IO, url: str, **kwargs) -> None:
    """  爬取一个 url 链接，将 html 包含的 href 写入文件 """
    res = await parse(url=url, **kwargs)
    if not res:
	return None
    async with aiofiles.open(file, "a") as f:
	for p in res:
	    await f.write(f"{url}\t{p}\n")
	logger.info("Wrote results for source URL: %s", url)

async def bulk_crawl_and_write(file: IO, urls: set, **kwargs) -> None:
    """ 异步的爬取多个 url 并写入文件 """
    async with ClientSession() as session:
	tasks = []
	for url in urls:
	    tasks.append(
		write_one(file=file, url=url, session=session, **kwargs)
	    )
	await asyncio.gather(*tasks)

if __name__ == "__main__":
    import pathlib
    import sys

    assert sys.version_info >= (3, 7), "Script requires Python 3.7+."
    here = pathlib.Path(__file__).parent

    """ 打开 urls 文件，将 url 读入 set """
    with open(here.joinpath("urls.txt")) as infile:
	urls = set(map(str.strip, infile))

    """ 打开存放提取出的 url 的文件 """
    outpath = here.joinpath("foundurls.txt")
    with open(outpath, "w") as outfile:
	outfile.write("source_url\tparsed_url\n")

    """ 运行异步任务 """
    asyncio.run(bulk_crawl_and_write(file=outpath, urls=urls))

并发编程

对于操作系统来说，一个任务就是一个进程（Process），比如打开一个浏览器就是启动一个浏览器进程，打开一个记事本就启动了一个记事本进程，打开两个记事本就启动了两个记事本进程，打开一个 Word 就启动了一个 Word 进程。

由于每个进程至少要干一件事，所以，一个进程至少有一个线程。当然，像Word这种复杂的进程可以有多个线程，多个线程可以同时执行，多线程的执行方式和多进程是一样的，也是由操作系统在多个线程之间快速切换，让每个线程都短暂地交替运行，看起来就像同时执行一样。当然，真正地同时执行多线程需要多核CPU才可能实现。

多进程

Unix/Linux 操作系统提供了 fork() 系统调用。它调用一次会返回两次，因为操作系统把当前进程复制了一份，分别在父进程和子进程内返回。

  • 子进程永远返回 0，父进程返回子进程的 ID。
  • 父进程可以有多个子进程，父进程要记录所有子进程的 ID。
import os

print("Process (%s) start ..." % os.getpid())

pid = os.fork()
if pid == 0:
    print("I'm child process (%s) and my parent is %s" % (os.getpid(), os.getppid()))
else:
    print("I'm (%s) just created a child process (%s)." % (os.getpid(), pid))

如果在 window 系统下执行多进程，可以使用 multiprocessing 模块。

from multiprocessing import Process

import os

def run_proc(name):
    print(f"Run child process {name} {os.getpid()} ...")

if __name__ == '__main__':
    print(f"Parent process {os.getpid()}")
    p = Process(target=run_proc, args=('test', ))
    print("Child process will start.")
    p.start()
    p.join()
    print('Child process end.')

如果想要启动大量的子进程，可以通过进程池的方式批量创建子进程。

from multiprocessing import Pool
import os, time, random

def task(name):
    print('Run task %s (%s)...' % (name, os.getpid()))
    start = time.time()
    time.sleep(random.random() * 3)
    end = time.time()
    print('Task %s runs %0.2f seconds.' % (name, (end - start)))

if __name__ == '__main__':
    print('Parent process %s.' % os.getpid())
    p = Pool(4)
    for i in range(5):
        p.apply_async(task, args=(i,))
    print('Waiting for all subprocesses done...')
    p.close()
    p.join()
    print('All subprocesses done.')

网络编程

Socket 是网络编程的一个抽象概念。通常我们用一个 Socket 表示“打开了一个网络链接”，而打开一个 Socket 需要知道目标计算机的 IP 地址和端口号，再指定协议类型即可。

TCP 编程

客户端

创建TCP连接时，主动发起连接的叫客户端，被动响应连接的叫服务器。

如果要创建一个基于 TCP 连接的 Socket，需要指定协议。

import socket

# AF_INET:指定 IPv4 协议	SOCK_STREAM:指定面向流的 TCP 协议
s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s.connect(('www.baidu.com', 80))

然后我们调用 send() 函数发送 HTTP 请求。

s.send(b'GET / HTTP/1.1\r\nHost: www.baidu.com\r\nConnection: close\r\n\r\n')

接着我们接收返回的数据，并将数据分离写入到文件中。

buffer = []
while True:
    d = s.recv(1024)
    if d:
        buffer.append(d)
    else:
        break
data = b''.join(buffer)

s.close()

header, html = data.split(b'\r\n\r\n', 1)
print(header.decode('utf-8'))
with open("baidu.html", 'wb') as f:
    f.write(html)

服务端

服务器进程首先要绑定一个端口并监听来自其他客户端的连接。如果某个客户端连接过来了，服务器就与该客户端建立Socket连接，随后的通信就靠这个Socket连接了。

所以，服务器会打开固定端口（比如80）监听，每来一个客户端连接，就创建该 Socket 连接。由于服务器会有大量来自客户端的连接，所以，服务器要能够区分一个 Socket 连接是和哪个客户端绑定的。一个 Socket 依赖4项：服务器地址、服务器端口、客户端地址、客户端端口来唯一确定一个 Socket。

import socket

s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

让服务端绑定本机地址 127.0.0.1 并且使用 9999 端口号。

s.bind(('127.0.0.1', 9999))

调用 listen() 方法监听端口，传入参数指定等待连接的最大数量。

s.listen(5)
print("Waiting for connection...")
