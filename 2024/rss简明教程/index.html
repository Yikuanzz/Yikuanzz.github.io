RSS简明教程

Sep 19, 2024
1370字

RSS

RSS（Really Simple Syndication） 是一种基于 XML 的内容聚合和发布格式。它允许用户在一个地方订阅和阅读来自多个网站的更新内容，而无需逐个访问这些网站。

主要特点：

  1. 内容聚合：RSS可以将来自不同网站的内容汇集到一个地方，方便用户统一阅读。
  2. 自动更新：订阅RSS源后，用户可以自动接收到最新的内容更新。
  3. 无广告干扰：RSS阅读器通常不会显示网站上的广告，提供更清洁的阅读体验。

使用方法：

  1. 找到RSS源：许多网站会提供RSS订阅链接，通常以图标或链接形式出现。
  2. 使用RSS阅读器：将RSS源链接添加到RSS阅读器中，如Feedly、Inoreader等，即可开始订阅和阅读内容。

这里以 Windows 端为例，接下来会介绍 Fluent RSS 阅读器，并且介绍如何进行 RSS 订阅获取自己想要知道的信息。

Fluent Reader

Fluent Reader 是一个外观简洁的本地跨平台新闻聚合器，能够随身携带用户所有感兴趣的资源。

添加订阅源

我们点击右上角边栏的设置齿轮，可以在上面添加订阅源，需要输入 URL 地址。

接下来，我们就需要获取订阅源了，获取订阅源的方式有两种：

  • 1.使用网站提供的 RSS 源，比如新闻网站、博客上提供的。
  • 2.使用 RSS 仓库，由开发者贡献的 RSS 订阅源。

这里介绍第一种获取订阅源的方式，下面介绍第二种订阅源的获取和使用。

以人民网举例，在要文快讯中有提供 RSS 的订阅源，我们可以使用它提供的 RSS 资源订阅消息。

然后，我们复制订阅源回到 Fluent Reader 中进行订阅，成功后我们就可以看到最新的新闻推送。

订阅源分组

当我们订阅的咨询比较多的时候，可以对订阅的信息进行分组，比如我这里订阅了时政频道、社会频道和教育频道，我想把他们都分在人民网这一个组里。

我可以在分组和排序中新建一个组，并把这些订阅源添加到组里。这样我就可以方便地对文章进行分类了。

偏好设置

RSS 的文章是通过抓取缓存在本地的，在偏好设置中我们可以设置文章抓取的频率，以及清理历史文章。

RSS Hub

一种更加便捷的方法是通过 RSS Hub 来去获取订阅源，它几乎能够从所有的内容中生成 RSS 提要。

  • 地址： https://rsshub.netlify.app/

示例：添加社媒

假设，我想添加豆瓣的每月推荐片单，我可以在其中找到写好的 URL 地址示例。

但是，注意需要做一些修改，比如举例中的是：https://rsshub.app/douban/recommended/tv

它不能直接粘贴到订阅源上去使用，而是需要更改地址中 rsshub.app 这个域名。这是官方提供的应用，但有时不是很稳定，更推荐使用一些公共的服务。

我们可以在指南中找到公共实例，选择可以使用的服务域名来更改原来的域名，比如我选择第一个 rsshub.rssforever.com ，替换之后的 URL 就是 https://rsshub.rssforever.com/douban/recommended/tv ，现在就可以放到订阅源去订阅了。

此外，示例外还提供了一些参数来去使用：

  • type 可以选择 tv 或者 movie

  • routeParams 可以筛选评分或者仅看有可播放片源的影片。

比如，我想看 movie 的榜单，而且仅看有可播放片源的影片，分数必须是 8 分的高分片。

我就可以这样写 /douban/recommended/movie/playable=1&score=8，然后再加上服务器域名就是订阅源了 https://rsshub.rssforever.com/douban/recommended/movie/playable=1&score=8 。
